# Render Deployment Guide for HPEvergreen Backend

## Prerequisites
- Docker Desktop installed and running
- Render account
- GitHub repository with this code

## Deployment Steps

### 1. Push to GitHub
```bash
git add .
git commit -m "Setup for Render deployment"
git push origin main
```

### 2. Deploy on Render
1. Go to [render.com](https://render.com)
2. Connect your GitHub account
3. Click "New" -> "Web Service"
4. Select your repository
5. Render will automatically detect the `render.yaml` file
6. Click "Create Web Service"

### 3. Environment Variables Setup
After deployment, you need to set these environment variables in Render dashboard:

#### Database Variables
- `MONGO_URI`: MongoDB connection string (Render will auto-generate this)
- `REDIS_URL`: Redis connection string (Render will auto-generate this)
- `REDIS_PASSWORD`: Redis password (Render will auto-generate this)

#### Email Variables (Optional)
- `SMTP_HOST`: Your SMTP server
- `SMTP_PORT`: SMTP port (usually 587)
- `SMTP_USER`: Your email
- `SMTP_PASS`: Your email password/app password

#### File Storage Variables (Optional)
- `CLOUDINARY_CLOUD_NAME`: Cloudinary cloud name
- `CLOUDINARY_API_KEY`: Cloudinary API key
- `CLOUDINARY_API_SECRET`: Cloudinary API secret

#### Frontend URL
- `FRONTEND_URL`: Your frontend application URL

### 4. Database Initialization
After deployment, you may need to initialize the database:
1. Go to Render dashboard -> Web Service -> Logs
2. Check if the database is connected
3. If needed, run the seed scripts through Render shell

## Troubleshooting

### Common Issues

#### 1. Database Connection Failed
**Symptoms**: Logs show MongoDB connection errors
**Solution**: 
- Check if MongoDB service is running in Render
- Verify MONGO_URI environment variable
- Wait a few minutes for MongoDB to fully start

#### 2. Redis Connection Failed
**Symptoms**: Logs show Redis connection errors
**Solution**:
- Check if Redis service is running in Render
- Verify REDIS_URL and REDIS_PASSWORD environment variables

#### 3. Application Not Starting
**Symptoms**: Web service shows "Failed" status
**Solution**:
- Check the logs in Render dashboard
- Verify all required environment variables are set
- Make sure the port is correctly set to 5000

#### 4. Health Check Failed
**Symptoms**: Service shows unhealthy status
**Solution**:
- Ensure `/health` endpoint is working
- Check if the application is fully started
- Verify the health check configuration

### Debug Steps
1. **Check Logs**: Always check Render logs first
2. **Verify Environment Variables**: Ensure all required variables are set
3. **Test Locally**: Try running with Docker locally first
4. **Check Service Dependencies**: Ensure MongoDB and Redis are running
